🏸 [채팅 및 매칭 관리] 상세 기획서

1. 채팅 목록 (Chat List)

사용자가 참여 중인 모든 대화방을 확인하는 화면입니다.

A. 와이어프레임 상세

상단 탭: [진행 중인 매칭] / [지난 매칭] 구분 (필요 시).

채팅방 리스트 아이템:

(좌) 매칭 대표 이미지 혹은 방장 프로필.

(중앙) 매칭 제목, 마지막 메시지 내용.

(우) 마지막 메시지 시간, 안 읽은 메시지 수(Red Badge).

(태그) [방장], [참여확정], [대기중] 등 나의 상태 표시.

B. 데이터 및 로직

데이터: room_id, last_message, unread_count, match_status.

로직: 최신 메시지가 온 순서대로 상단에 정렬됩니다. 매칭이 종료된 채팅방은 별도 섹션으로 분리하거나 하단으로 배치합니다.

2. 채팅 상세 (Chat Detail)

실제 대화가 이루어지는 공간이며, 매칭의 핵심 정보가 상단에 고정됩니다.

A. 와이어프레임 상세

상단 고정 정보 탭 (Sticky Info):

매칭 일시, 장소명, 현재 인원 정보 요약.

[지도보기] 버튼 포함.

메시지 영역:

시스템 메시지: "○○님이 입장하셨습니다", "방장이 매칭을 확정했습니다" 등.

사용자 메시지: 좌(타인)/우(나) 말풍선 구분. 닉네임과 프로필 사진 노출.

하단 입력바: 텍스트 입력창, 사진 첨부(+), 전송 버튼.

B. 데이터 및 로직

실시간성: WebSocket 혹은 Firebase Realtime DB를 통한 실시간 메시지 송수신.

시스템 메시지 트리거: * 방장이 게스트의 신청을 '수락'하는 순간 시스템 메시지와 함께 해당 유저가 채팅방에 자동 입장됩니다.

매칭 정보 연동: 상단 탭 클릭 시 '매칭 상세 페이지'로 다시 이동할 수 있어야 합니다.

3. 방장 전용 매칭 관리 (Host Management)

방장이 신청자를 관리하고, 모임 후 매너를 체크하는 관리자 페이지입니다.

A. 와이어프레임 상세

섹션 1: 신청자 관리 (Pre-match):

신청한 유저들의 프로필 리스트 (닉네임, 급수, 후기 점수, 한마디).

각 유저 우측에 [수락], [거절] 버튼.

섹션 2: 참여자 관리 (Post-match / 지각·노쇼 체크):

확정된 인원 리스트.

모임 시간 이후 활성화되는 [출석 완료], [지각], [노쇼] 체크 박스 혹은 버튼.

섹션 3: 매칭 상태 제어:

[매칭 취소하기]: 부득이한 사정으로 방장이 모임을 폭파할 때 사용.

[매칭 마감하기]: 정원이 차지 않았더라도 추가 신청을 막고 싶을 때 사용.

B. 데이터 및 로직

수락/거절 로직: * 방장이 '수락' 시 해당 유저는 confirmed_participants 리스트로 이동하며 채팅방에 초대됩니다.

'거절' 시 해당 유저에게는 거절 알림이 가며 리스트에서 사라집니다.

지각/노쇼 페널티: * 방장이 '노쇼'로 체크하고 [확인]을 누르면 해당 유저의 신뢰도 점수가 즉시 차감되며, 해당 이력이 유저 DB에 기록됩니다.

후기 작성 유도: * 모임 종료 예정 시간 2시간 후, 방장과 참여자 모두에게 '오늘 매칭은 어떠셨나요? 후기를 남겨주세요'라는 알림을 발송합니다.
