🏸 [매칭 만들기] 화면 상세 기획서

1. 와이어프레임 상세 (Wireframe Layout)

사용자가 위에서 아래로 물 흐르듯 입력할 수 있는 싱글 페이지 폼 구조입니다.

A. 상단 헤더 (Header)

(좌측) 취소 버튼: 입력 중인 내용을 파기하고 홈으로 이동 (확인 팝업 필요).

(중앙) 타이틀: "매칭 만들기".

(우측) 등록 버튼: 모든 필수 항목 입력 시 활성화.

B. 기본 정보 섹션 (Basic Info)

매칭 제목: (Input) 예: "망원동에서 가볍게 한 판 하실 분!".

상세 설명: (Textarea) 모임에 대한 자세한 안내 (준비물, 분위기 등).

C. 일시 및 장소 섹션 (Time & Location)

날짜 선택: (Date Picker) 오늘부터 최대 2주 후까지 선택 가능.

시간 및 소요 시간: - 시작 시간 (Time Picker).

운동 시간 (Dropdown: 1시간, 1.5시간, 2시간 ... 4시간).

장소 검색: (Search Input) 클릭 시 지도 API 검색 화면으로 연결. 장소 명칭과 주소 노출.

D. 참여 조건 및 규칙 (Rules & Target)

모집 인원: (Stepper) 최소 2명 ~ 최대 12명 (방장 본인 포함 여부 명시).

희망 급수: (Multi-Select Chips) 초심, D, C, B, A 중 선택 (중복 선택 가능).

성별 제한: (Radio/Toggle) 성별 무관, 남성만, 여성만.

비용 분담: (Dropdown) 1/N 정산, 방장 부담, 게스트 부담.

E. 사진 등록 (Optional)

이미지 업로드: 배드민턴장 사진이나 팀 로고 등 등록 (최대 3장).

3. 핵심 비즈니스 로직 (Business Logic)

A. 입력값 검증 (Validation)

과거 시간 제한: 현재 시간보다 이전의 날짜나 시간으로는 매칭을 생성할 수 없습니다.

필수 항목 체크: 제목, 설명, 장소, 시간, 인원 등 필수 항목이 하나라도 누락되면 [등록] 버튼은 비활성화(Disabled) 상태를 유지합니다.

인원 수 제한: 배드민턴 코트 특성상 한 코트당 최대 6~8명이 적당하므로, 너무 많은 인원(예: 20명 이상)은 경고 메시지를 노출합니다.

B. 등록 및 자동 처리

방장 자동 등록: 매칭을 생성한 유저는 해당 매칭의 host로 자동 등록되며, current_participants 수는 자동으로 1이 됩니다.

채팅방 자동 생성: 등록 완료 즉시 해당 match_id와 연결된 전용 채팅방이 백엔드에서 생성됩니다.

관심 지역 알림: 매칭이 등록되면 해당 지역을 '관심 지역'으로 설정한 유저들에게 "내 주변에 새로운 매칭이 올라왔어요!" 푸시 알림을 발송합니다.

C. 예외 처리

중복 시간대 경고: 방장이 이미 같은 시간대에 다른 매칭을 만들었거나 참여 중인 경우 "이미 해당 시간에 다른 일정이 있습니다"라는 안내 문구를 띄워줍니다.

중도 이탈 방지: 내용을 입력하던 중 '취소'나 '뒤로가기'를 누르면 "작성 중인 내용이 사라집니다. 취소하시겠습니까?"라는 확인 모달을 띄웁니다.
