
## 1. 와이어프레임 레이아웃 (Layout)

화면을 위에서 아래로 흐르는 4개의 섹션으로 구분합니다.

* **[상단 헤더]**
* (좌) 뒤로가기 버튼 / (우) 공유하기, 신고하기 버튼.


* **[섹션 1: 방장 정보 및 제목]**
* 방장 프로필 이미지(원형), 닉네임, **매너 점수(별점/해시태그)**.
* 매칭 게시글 제목 (굵은 텍스트).


* **[섹션 2: 핵심 매칭 카드]** - 별도 박스 처리
* **일시:** 2026.03.10(화) 19:00 ~ 21:00 (2시간).
* **장소:** ○○ 배드민턴장 (클릭 시 지도 앱 연동).
* **인원:** 현재 참여 3명 / 정원 4명 (**대기 중인 인원: 2명**).
* **비용:** 1/N 정산 (예상 인당 5,000원).


* **[섹션 3: 상세 내용 및 조건]**
* **희망 급수:** C조 이상 (초심 불가).
* 방장의 상세 설명 (텍스트).
* **확정된 참여자 리스트:** 프로필 아이콘들의 나열 (클릭 시 해당 유저 프로필 팝업).


* **[섹션 4: 하단 고정 바]** - 스크롤과 상관없이 하단에 고정
* 상태에 따라 변하는 **[액션 버튼]**.

## 2. 데이터 명세 (Data Specification)

프론트엔드와 백엔드가 주고받을 데이터 필드를 정의합니다.

| 항목명 | 변수명(예시) | 타입 | 필수 여부 | 설명 |
| --- | --- | --- | --- | --- |
| **매칭 ID** | `match_id` | Long (PK) | 필수 | 시스템 식별값 |
| **방장 ID** | `host_id` | Long (FK) | 필수 | 유저 테이블 참조 |
| **매칭 제목** | `title` | String | 필수 | 최대 50자 |
| **모집 상태** | `status` | Enum | 필수 | 모집중(RECRUITING), 마감(CLOSED), 종료(FINISHED) |
| **모집 인원** | `max_people` | Integer | 필수 | 2~12명 사이 |
| **현재 확정 인원** | `current_people` | Integer | 필수 | 참여 확정된 인원 수 |
| **장소 정보** | `location` | Object | 필수 | 위도, 경도, 장소명, 주소 포함 |
| **목표 급수** | `target_level` | Enum | 필수 | A, B, C, D, 초심, 무관 중 선택 |
| **비용 분담** | `cost_policy` | String | 필수 | 1/N, 방장부담 등 텍스트 |

## 3. 핵심 비즈니스 로직 (Business Logic)

이 페이지의 가장 핵심인 **'하단 버튼'의 상태 변화** 로직입니다. 유저의 역할과 매칭 상태에 따라 버튼이 다르게 보여야 합니다.

### A. 버튼 상태 로직 (User Role & Status)

1. **방장(Host)인 경우:**
* 참여 신청자가 있을 때: **[신청자 관리(N)]** 버튼 노출 (클릭 시 수락/거절 화면으로 이동).
* 채팅방 가기 버튼 노출.


2. **일반 유저(Guest) - 아직 신청 전:**
* 정원이 남았을 때: **[참여 신청하기]** 버튼.
* 정원이 찼을 때: **[대기 신청하기]** 버튼 (대기열 시스템).


3. **일반 유저(Guest) - 신청 후 수락 대기 중:**
* **[수락 대기 중... (취소하기)]** 버튼 (비활성화 느낌의 디자인).


4. **일반 유저(Guest) - 수락 완료(참여 확정):**
* **[채팅방 입장하기]** 버튼으로 변경.



### B. 페널티 및 후기 로직

* **모임 종료 후 (+2시간 뒤):** 버튼이 **[후기 남기기]**로 자동 변경됩니다.
* **지각/노쇼 신고:** 방장이 채팅방 내 관리 메뉴에서 '노쇼' 체크 시, 해당 유저의 `penalty_count`가 +1 되고 프로필 점수에 즉시 반영됩니다.

### C. 대기열 시스템 (Queue Logic)

* 확정 인원 중 누군가 '참여 취소'를 하면, **대기열 1번 유저**에게 "참여가 가능해졌습니다! 10분 내로 확정해 주세요."라는 Push 알림이 발송됩니다. (10분 내 미확정 시 2번으로 기회 이전)
