🏸 [홈 / 매칭 목록] 화면 상세 기획서

1. 와이어프레임 상세 (Wireframe Layout)

화면을 상단에서 하단으로 5개의 주요 영역으로 나눕니다.

A. 상단 헤더 (Top Bar)

(좌측) 지역 선택 드롭다운: 사용자가 설정한 관심 지역 1, 2를 탭으로 전환하거나 드롭다운으로 선택.

(우측) 알림 아이콘: 새로운 매칭 수락 알림, 채팅 알림 등이 있을 때 레드 닷(Red Dot) 표시.

B. 날짜 및 퀵 필터 (Date & Quick Filter)

날짜 피커: [오늘], [📅 달력 아이콘] 형태의 가로 스크롤 버튼.

필터 칩: [급수 선택], [시간대], [성별], [모집 중만 보기] 등 클릭 시 바텀 시트(Bottom Sheet)가 올라오는 형태.
  - 급수선택: 클릭시 'A', 'B', 'C', 'D', '초심' 복수선택 가능하도록
  - 시간대: 오전, 오후 선택
  - 성별: 남자, 여자, 상관없음

C. 매칭 카드 리스트 (Matching Card List) - 세로 스크롤

각 카드(아이템)는 다음 요소를 포함합니다:

상태 라벨: [모집중], [종료], [대기신청] (색상으로 구분).

제목: "일요일 오후 초심~D조 즐겁게 치실 분!" (최대 2줄).

매칭 정보: - 🕒 시간: 19:00 ~ 21:00 (2h)

📍 장소: 마포구 성미산 체육관

👥 인원: 3/4 명 (현재/총원)

방장 정보 및 신뢰도: 닉네임 + 별점(4.8) + 대표 해시태그(#매너왕).

급수 배지: D, C 등 눈에 띄는 컬러 배지.

D. 플로팅 버튼 (Floating Action Button)

[+] 버튼: 화면 우측 하단 고정. 클릭 시 '매칭 만들기' 화면으로 이동.

E. 하단 탭바 (Bottom Navigation)

[홈(활성)], [채팅], [마이페이지] 3개 메뉴로 구성.

3. 핵심 비즈니스 로직 (Business Logic)

A. 리스트 필터링 및 정렬 로직

기본 노출: 사용자가 마지막으로 선택했거나 첫 번째로 등록한 **'관심 지역'**의 매칭을 최신순으로 먼저 보여줍니다.

지역 전환: 상단 지역 탭 클릭 시 해당 지역 데이터로 즉시 새로고침됩니다.

복합 필터: 날짜와 급수를 동시에 선택하면 AND 조건으로 필터링된 결과만 노출합니다.

Empty State: 필터 조건에 맞는 매칭이 없을 경우 "해당 조건의 매칭이 없습니다. 새로운 매칭을 만들어보세요!" 문구와 [매칭 만들기] 버튼을 중앙에 크게 노출합니다.

B. 사용자 인터랙션 로직

새로고침 (Pull-to-Refresh): 리스트를 아래로 당기면 최신 매칭 정보를 서버에서 다시 가져옵니다.

무한 스크롤 (Pagination): 한 번에 20개씩 로딩하며, 스크롤이 끝에 닿으면 다음 20개를 로드합니다.

중복 신청 방지: 이미 신청했거나 내가 방장인 매칭 카드는 리스트에서 '신청 완료' 또는 '내 모임' 배지를 별도로 표시하여 혼동을 방지합니다.

C. 실시간 상태 업데이트

누군가 참여를 확정하여 정원이 차면, 서버는 실시간(Websocket 등) 또는 주기적으로 상태를 전송하여 해당 카드의 상태를 **[모집중] → [대기신청]**으로 즉시 변경합니다.
