## Sprint 4: 후기 및 패널티 (2주)

### 목표
후기 작성·조회, 패널티 부여·조회가 동작한다.

### 백엔드
| 작업 | 상세 | 백로그 ID |
|------|------|------------|
| 매칭 종료 | 방장 수동 `FINISHED` 전환 + **스케줄러 자동 종료**(모임 시작 후 예: 24시간 경과 시 CLOSED→FINISHED 등, 정책은 `docs/요구사항분석.md`) | B5-1 전제 |
| Review 엔티티 | 평가자, 피평가자, 매칭, 점수, 해시태그(고정 태그는 Enum/참조 테이블 검토) | B5-1, B5-2 |
| 후기 API | 작성, 조회(**상호 익명·공개 시점** 규칙 반영, `docs/요구사항분석.md`) | B5-1, B5-3 |
| 평점 갱신 | `User.rating_score` — **가상 프라이어·가중 갱신**(단순 평균 지양) | B5-2 |
| Penalty 엔티티 | 노쇼, 지각 | B5-4 |
| 패널티 API | 방장 체크, 이력 조회 | B5-4, B5-5 |
| 단계별 제재 | 스트라이크(또는 penalty_points)에 따른 **주의·참여 제한·정지·퇴출**; `participation_banned_until` / `suspended_until` 갱신, 참여 신청 API 검증 | `docs/요구사항분석.md` |
| 프로필 연동 | 제한·정지 **만료 시각**(또는 남은 시간) 노출 | B5-5 |

### 스프린트 용량 점검 (2주)
- **자동 종료 스케줄러·익명 공개 규칙·평점 수식**은 범위가 늘어나지만, 모두 **단일 인스턴스 배치 + 도메인 서비스 분기** 수준이면 2주 스프린트 내 포함 가능한 편이다.
- **권장 우선순위**: (1) FINISHED 수동 + 자동 종료, (2) 후기 CRUD + 조회 마스킹, (3) 평점 가중 갱신, (4) 패널티 CRUD, (5) **누적 제재·User 상태**는 스코프 압박 시 Sprint 4 말미 또는 다음 스프린트로 미룰 수 있다.
- 인덱스·fetch join 등 성능 작업은 후기 목록 조회와 함께 **최소한** 반영한다.

### 프론트엔드
| 작업 | 상세 | 백로그 ID |
|------|------|------------|
| 후기 작성 화면 | 3단계 좋아요, 점수, 해시태그 선택 | B5-1, B5-2 |
| 후기 조회 | 유저별 후기 목록 | B5-3 |
| 패널티 체크 | 방장의 노쇼/지각 체크 UI | B5-4 |
| 패널티 이력 | 유저 프로필에 패널티 표시 | B5-5 |

### 완료 조건
- [ ] 모임 종료 후 후기 작성 가능(방장 수동 종료 및 **자동 종료**로 FINISHED 도달)
- [ ] 유저 프로필에서 후기·패널티 확인 가능
- [ ] 후기 조회 시 **상호 익명·공개 시점** 정책이 API에 반영됨(`docs/요구사항분석.md`)
- [ ] 평점은 **프라이어 반영 갱신**으로 산정됨
- [ ] 패널티 **단계별 제재**(`participation_banned_until` / `suspended_until` 등) 및 참여 신청 검증 동작
- [ ] 프로필에서 **제한·정지 만료 시각**(또는 남은 시간) 확인 가능

## Sprint 7: 마이페이지·운영·계정 API (2주) — 후속

> **백엔드 상세 할일**: `docs/Sprint7-할일.md`

### 목표
마이페이지 요약과 **별도 화면**을 정합시킨다. **`GET /api/users/me`**에는 **개설·참여 매칭 수(집계)** 만 두고, **매칭 내역·후기(작성함/대기)** 는 **별도 API·별도 라우트**로 둔다. **B9-5·B9-6·B9-7** 은 **본 스프린트 제외** (`docs/Sprint7-할일.md` Step 0 합의).

### 백엔드
| 작업 | 상세 | 백로그 ID |
|------|------|------------|
| **`GET /api/users/me`** | `ProfileResponse`에 **개설·참여 매칭 수**만 추가 (P1·P2). 목록·프리뷰 없음 | B9-1 |
| **매칭 내역 별도 화면** | 개설 목록·참여 목록 — `GET /api/matches` 쿼리 확장 또는 전용 경로 (`Sprint7-할일.md` Step 2) | B9-2, B9-3 |
| **후기 별도 화면** | **내가 쓴 후기**·**작성 대기** 조회 API (`Sprint7-할일.md` Step 3) | B9-4 |
| ~~공지·고객센터·계정~~ | **스프린트 제외** | B9-5 ~ B9-7 |

### 프론트엔드
| 작업 | 상세 | 백로그 ID |
|------|------|------------|
| 마이페이지 (`/profile/me` 등) | **`GET /api/users/me`** 로 **숫자 집계**만 표시 | B9-1 |
| 매칭 내역 화면 | Step 2 API로 **개설/참여 각각 목록** | B9-2, B9-3 |
