# 플러터 개발 환경 설정

## 사전 요구사항
- Flutter SDK: 3.x (Stable)
- Xcode (iOS 개발용) / Android Studio (Android 개발용)
- CocoaPods (iOS 라이브러리 관리)

## 환경 구분 (Flavor/Entry point)
플러터는 별도의 `--target`이나 `Define` 옵션을 통해 환경을 구분합니다.

| 환경 | 기본 API 베이스 URL (`API_BASE_URL` 미지정 시) | 참고 |
|------|-----------------------------------------------|------|
| **dev** (`APP_ENV=dev`) | `http://localhost:8080` | 플랫폼별로 아래 표에서 URL을 `--dart-define`으로 덮어쓸 수 있음 |
| **prod** (`APP_ENV=prod`) | `https://api.minton-match.com` | 릴리스 빌드 시 `APP_ENV=prod` 권장 |

앱 코드: `AppEnvironment.appConfig` (`lib/core/env/app_environment.dart`, 내부적으로 `AppConfig.fromDartDefines()`).

## 로컬 백엔드 연동 시 호스트 (플랫폼별)

맥/PC에서 `localhost:8080`으로 띄운 API에 앱이 붙을 때, **실행 환경마다 접속 주소가 다릅니다.** 네트워크 오류 시 아래를 먼저 확인하세요.

| 실행 환경 | 권장 Base URL 예시 | 비고 |
|-----------|-------------------|------|
| **Android 에뮬레이터** | `http://10.0.2.2:8080` | 에뮬레이터의 `localhost`는 에뮬레이터 자신을 가리킴. 호스트 PC는 `10.0.2.2` |
| **iOS 시뮬레이터** | `http://localhost:8080` | 호스트와 동일 네트워크 스택을 쓰는 경우가 많아 `localhost` 사용 가능 |
| **실제 기기** | `http://<맥의 LAN IP>:8080` (예: `192.168.x.x`) | 기기와 개발 PC가 **같은 Wi‑Fi**에 있어야 함. 방화벽에서 해당 포트 허용 |

`API_BASE_URL` 또는 `--dart-define` 값은 위에 맞게 dev 프로필에서 바꿔 쓰면 됩니다.

## 외부 API 및 소셜 로그인 설정
플러터는 플랫폼별(iOS, Android) 네이티브 설정이 필요합니다.

### 1. Kakao 로그인 (OAuth)
- **인가 URL `client_id`**: `KAKAO_CLIENT_ID` — 보통 **REST API 키**(Next `NEXT_PUBLIC_KAKAO_CLIENT_ID`·백엔드와 동일 권장). 예전 이름 `KAKAO_NATIVE_KEY`도 읽음.
- **모바일 리다이렉트**: `KAKAO_NATIVE_APP_KEY`(콘솔 **네이티브 앱 키**)만 넣고 `OAUTH_REDIRECT_URI`를 비우면 앱이 `kakao{키}://oauth` 를 사용합니다. REST API 키 수정 화면에는 `http(s)://`만 넣을 수 있는 경우가 많으므로, 콘솔 **플랫폼 Android/iOS** 쪽 가이드를 따르세요.
- **딥링크**: `kakao…` 스킴을 쓰면 [OAuth-환경변수.md](./OAuth-환경변수.md) 및 `docs/snippets/kakao-deeplink-*.xml|plist` 로 Android/iOS를 맞춥니다. 기본 커밋에는 `com.mintonmatch.app` 스킴도 포함되어 있습니다.

### 2. 환경 변수 관리 (`--dart-define`)
컴파일 타임에 주입합니다. 코드는 `String.fromEnvironment`로 읽습니다 (`AppConfig`).

| 키 | 기본값 | 설명 |
|----|--------|------|
| `APP_ENV` | `dev` | `dev` → 개발 기본 URL, `prod` / `production` / `release` → 운영 기본 URL |
| `API_BASE_URL` | (비어 있음) | 비어 있으면 `APP_ENV`에 따른 기본 URL 사용. 지정 시 **항상 이 값** 사용 |
| `KAKAO_CLIENT_ID` | (비어 있음) | 카카오 인가 `client_id`(보통 REST API 키). `KAKAO_NATIVE_KEY`도 동일 용도(하위 호환) |
| `KAKAO_NATIVE_APP_KEY` | (비어 있음) | 카카오 **네이티브 앱 키**. 있으면 `OAUTH_REDIRECT_URI` 비울 때 `kakao{키}://oauth` |
| `OAUTH_REDIRECT_URI` | (비어 있음) | 비우면 위 규칙·기본 스킴 사용. Web은 `http://localhost:3000/oauth/callback` 등 가능 |

여러 개를 넘길 때는 각각 `--dart-define=KEY=value`를 반복합니다.

## 실행 및 빌드 명령어

### 로컬 실행 (디버그 · dev 기본값)
```bash
flutter run --debug
```

### Android 에뮬레이터에서 호스트 PC의 로컬 API
```bash
flutter run --debug --dart-define=API_BASE_URL=http://10.0.2.2:8080
```

### iOS 시뮬레이터 · 로컬호스트
```bash
flutter run --debug --dart-define=API_BASE_URL=http://localhost:8080
```

### 실제 기기 (LAN IP 예시)
```bash
flutter run --debug --dart-define=API_BASE_URL=http://192.168.0.12:8080
```

### 운영 API + 릴리스 모드
```bash
flutter run --release --dart-define=APP_ENV=prod
```

### 운영 URL을 명시적으로 지정
```bash
flutter run --release --dart-define=APP_ENV=prod --dart-define=API_BASE_URL=https://api.minton-match.com
```

### 카카오 키 포함 예시 (모바일: 네이티브 앱 키로 리다이렉트 자동)
```bash
flutter run --debug \
  --dart-define=API_BASE_URL=http://10.0.2.2:8080 \
  --dart-define=KAKAO_CLIENT_ID=REST_API_키 \
  --dart-define=KAKAO_NATIVE_APP_KEY=네이티브_앱_키
```
`dart_defines.oauth.local.env` 사용을 권장합니다. ([OAuth-환경변수.md](./OAuth-환경변수.md))

### APK/IPA 빌드 시
```bash
flutter build apk --release --dart-define=APP_ENV=prod --dart-define=KAKAO_CLIENT_ID=... --dart-define=KAKAO_NATIVE_APP_KEY=...
flutter build ios --release --dart-define=APP_ENV=prod --dart-define=KAKAO_CLIENT_ID=... --dart-define=KAKAO_NATIVE_APP_KEY=...
```
